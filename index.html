AMP - Accelereted Mobile Page

  • Open-source initiative aiming to make the web better for all
  • Speeding up the delivery of content through the use of stripped down code known as AMP HTML
  • Make websites fast, beautiful and high-performing across devices
  • AMP is a way to build web pages for static content

AMP HTML

AMP HTML is HTML with some restrictions for reliable performance.

  • Start with the <!doctype html>
  • a top-level <html ⚡> tag
  • <script async src="https://cdn.ampproject.org/v0.js"></script> tag
  • <link rel="canonical" href="$SOME_URL"> tag inside their <head>
  • Contain the AMP boilerplate code in their <head> tag.
    • This code itself is to hide the page until it is fully rendered, and then fade it in to give a higher perceived performance metric.

Common Attributes

layout

AMP provides a set of layouts that specify how an AMP component behaves in the document layout. You can specify a layout for a component by adding the layout attribute with one of the supported layout values for the element.

nodisplay, fixed, responsive, fixed-height, fill, container, flex-item

width

height

media

All AMP elements support the media attribute. The value of media is a media query. If the query does not match, the element is not rendered and its resources and potentially its child resources will not be fetched. If the browser window changes size or orientation, the media queries are re-evaluated and elements are hidden and shown based on the new results.

noloading

The noloading attribute indicates whether the "loading indicator" should be turned off for this element. Many AMP elements show a "loading indicator", which is a basic animation that shows that the element has not yet fully loaded.

placeholder

The placeholder attribute indicates that the element marked with this attribute acts as a placeholder for the parent AMP element.

fallback

You can specify the fallback attribute on an element to indicate the fallback behavior:

  • for any element the browser doesn’t support
  • if the content fails to load
  • if the image type is unsupported

You can set the fallback attribute on any HTML element, not just AMP elements. If specified, the fallback element must be a direct child of the AMP element.

AMP JS

  • The AMP JS library ensures the fast rendering of AMP HTML pages.
  • AMP allows only asynchronous JavaScript.

AMP Cache

The Google AMP Cache can be used to serve cached AMP HTML pages.

two AMP Cache providers

  • Google AMP Cache
  • Cloudflare AMP Cache

As a publisher, you don't choose an AMP Cache, it's actually the platform that links to your content that chooses the AMP Cache (if any) to use.

Caching is a core part of the AMP ecosystem. Publishing a valid AMP document automatically opts it into cache delivery.

How does my AMP page get cached?

  • Platform discovery
  • Cache URL request
    • Cloudflare AMP Cache URL example:
      https://amp.cloudflare.com/c/foo.com/amp_document.html
    • Google AMP Cache URL example:
      https://foo-com.cdn.ampproject.org/c/s/foo.com/amp_document.html
  • Publisher addition
