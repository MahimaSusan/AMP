AMP - Accelereted Mobile Page

Open-source initiative aiming to make the web better for all

speeding up the delivery of content through the use of stripped down code known as AMP HTML

Make websites fast, beautiful and high-performing across devices

AMP is a way to build web pages for static content

AMP HTML

AMP HTML is HTML with some restrictions for reliable performance.

  • Start with the <!doctype html>
  • a top-level <html ⚡> tag
  • <script async src="https://cdn.ampproject.org/v0.js"></script> tag
  • <link rel="canonical" href="$SOME_URL"> tag inside their <head>
  • Contain the AMP boilerplate code in their <head> tag.
    • This code itself is to hide the page until it is fully rendered, and then fade it in to give a higher perceived performance metric.

Attributes

Not allowed

  • Conditional HTML comments
  • Attributes starting with ‘on’
  • XML related attributes
  • Internal AMP attributes prefixed with i-amp-

Classes and IDs

Disallowed

Classes and IDs with -amp- and i-amp-

Stylesheets

  • allowed
    • @font-face, @keyframes, @media, @support
  • Disallowed
    • @import
    • !important
  • Custom styles need to be added inside <style amp-custom> tag

CSS properties that are permanently banned

  • behavior
  • moz-binding
  • transform
  • overflow may not be styled as ‘auto’ or ‘scroll’

Actions and events in AMP

The on attribute is used to install event handlers on elements. The events that are supported depend on the element. The value for the syntax is a simple domain-specific language of the form: eventName:targetId[.methodName[(arg1=value, arg2=value)]]
